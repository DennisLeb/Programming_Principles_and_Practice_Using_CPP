[1] What is the difference between a declaration and a definition?
A declaration is used to reserve a name for an object, without creating it in memory.
Definition reserves a name AND allocates memory for an object, initialzing it to some value, or in the case of functions, a computation.

[2] How do we syntactically distinguish between a function declaration and a function definition?
A function declaration needs only return type, name and arguments. A declarations ends with ;.
	Example:
		void my_func(int i);

A definition requires a function block in addition to its declaration elements.
	Example:
		void my_func(int i) { std::cout << "i == " << i; }

A function can be declared before providing a definition for it.

[3] How do we syntactically distinguish between a variable declaration and a variable definition?
A variable declaration only provides data type and name. The extern keyword can be used to signify a declaration without definition to readers.
	Example:
		extern string s;

For a variable definition you need some sort of assignment syntax (=, (), {})
	Example:
		int i = 0;

Variables can also be declared before they're initialized (defined), though this is bad practice because it can cause unexpected bugs.

[4] Why can't you use the functions in the calculator program from Chapter 5 without declaring one or more of them first?
Since these functions's call order is cyclical (recursive), there isn't a definition order that would allow each of them to be defined before being called.

A "forward declaration" for expression() was needed to tell the compiler (when entering primary()) that there is indeed a function by the name of expression() and
to make it look for its definition.

[5] Is int a; a definition or just a declaration?
This is a declaration.

[6] Why is it a good idea to initialize variables as they are declared?
Because if they're compared in any following statements before receiving a value, it will cause an error which may lead to unnecessary confusion.

[7] What can a function declaration consist of?
Return type, name and arguments, ending with a ;. An additional function block would make it a definition.

Arguments also don't need names in a declaration, meaning one could look like this:
	int my_func(int, int);

and later be defined as this:
	int my_func(int a, int b) { return a + b; }

[8] What is the suffix return type notation, and why might you use it?
The suffix return type notation look as follows:
	auto my_func(int a, int b) -> int;

It's useful if it makes a function more readable. Another example:
	auto find_string_index(string s, vector<string> s_vec) -> int;

[9] What does good indentation do?
It organizes code (or text in general) in a way, that makes it much more readable and pleasant to look at.

[10] What is the scope of a declaration?
It reaches from the point of its declaration to the respective closing bracket }.
All names in a scope can be seen from within scopes nested within it.

[11] What kinds of scopes are there? Give an example of each.
- The global scope:
	the area of text outside any other scope
- A module scope:
	the area of text within a module
_ A namespace scope:
	a named scope nested within the global scope or in another namespace
- A class scope:
	the area of text within a class
- A local scope:
	between {...} braces of a block or in a function argument list
- A statement scope:
	as in a for-statement

[12] What is the difference between a class scope and a local scope?
Class scope objects are accessible to all other member variables and functions of that class, as well as variables of the class type.
Local scope objects are only accessible to itself and other scopes nested within.

This means that, if you declare a class variable, it will be in-scope for all class objects.

[13] Why should a programmer minimize the number of global variables?
Because most variables aren't needed in every other scope, making it inefficient.

Though it is possible to define new local variables with the name of a global one, it can get confusing very quickly.
Too many global variables take up too many names.

[14] What is the difference between pass-by-value and pass-by-reference?
When a function argument variable is passed-by-value a copy of it is created each time, demanding more resuorces from your machine.
Modifications to that copy won't be saved to that variable outside that function's scope.

Passing such an argument by-reference means passing the memory adress of a variable outside the function, instead of creating a new copy.
Modifications to such arguments will carry over outside of the local scope.

[15] What is the difference between pass-by-reference and pass-by-const-reference?
Passing-by-const-reference has the benefit of better performance through not creating a copy, without the risk of performing unwanted operations on the referenced object.

[16] What is swap()?
swap() is a funtion that takes in two parameters and swaps their values. To do this the parameters need to be passed-by-reference, since functions may only return one value.

[17] Would you ever define a function with a vector<double> as a by-value parameter?
Most likely not. For larger data amounts this would be far too expensive and even if the vector wasn't very large, there's just no real benefit.
Also, if you wanted to avoid modifying the data by passing-by-value, you might as well pass-by-const-reference to achieve the same result, but more efficiently.

[18] Give an example of undefined order of evaluation. Why can undefined order of evaluation be a problem?
Example:
	x = i++ + i;

This is bad code because the result of this expression may vary from compiler to compiler. Its ambiguity makes it a problem.

[19] What do x&&y and x||y, respectively, mean?
Both are logical operators which return a boolean value. They're called "logical and" and "logical or".

Logical and returns true if both x and y return true.
Logical or returns true if either x or y returns true.

[20] Which of the following is standart-conforming C++: functions within functions(1), functions within classes(2), classes within classes(3), classes within functions(4)?
(1) Function nesting is NOT legal in C++. (Instead use a lambda)
(2) Member functions are legal. They're quite common and useful.
(3) Nested classes are legal, but they should be reserved for complicated cases. The ideal is to keep classes small and simple.
(4) Local classes are legal.

[21] What goes into an activation record?
When a function is called, all of its parameters and local variables are stored in the activation record.

[22] What is a call stack and why do we need one?
The call stack keeps the activation records of every called function during run-time, until they return.
This is necessary for function calls within functions. The call stack remembers the outer function's object's states until the inner function returns.

[23] What is the purpose of a namespace?
A namespace organizes code under a common name. This avoids object names clashing and provides further logical division of code.

[24] How does a namespace differ from a class?
Namespaces allow you to organize classes, functions and other data into a named part of a program without defining a type.

[25] What is a using declaration?
using declaraions make namespaces accessible to a program without having to specify the namespace in every instance of use.
It's a way of telling the compiler "if you don't find this name here, look in this namespace".

[26] Why should you avoid using directives in a header?
Because indirectly using a namespace may lead to confusion if, for example, a named object from that namespace clashes with another local object.
You should always be keenly aware of what using directives you're implementing.

[27] What is namespace std?
namespace std refers to the C++ standard library namespace, which provides many useful functions or other objects for your use.
